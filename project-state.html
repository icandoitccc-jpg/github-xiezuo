D-002 · 2026-06-26
为什么放弃优先开发大渠道（REI）？
REI 进场门槛极高（EDI、认证、提前备货），公司当前资源不支持。优先找 40 家小B客户验证产品和触达机制，成本更低、反馈更快。
D-003 · 2026-06-29
为什么把报告部署到 GitHub Pages 而不是 CloudStudio / paste.rs？
CloudStudio 域名被 ChatGPT/Gemini 抓取工具拒绝；paste.rs 结构让 ChatGPT 无法读到正文。GitHub Pages 是标准静态 HTML，Gemini 可以直接读取，ChatGPT 粘贴可用。团队报告共享需要稳定公网链接。
D-004 · 2026-06-30
为什么不把"北美开发"和"AI协作"分开做？
两者是互相咬合的双引擎。北美开发提供真实业务压力（不能只在理论层面讨论），AI协作系统从真实任务中迭代。任何一方单独推进都会脱离现实或脱离验证机会。
D-005 · 2026-06-30
Context Card → Project Brain → Handoff → MCL，为什么不断推翻？
每一版都在解决更底层的问题。Context Card 解决信息同步 → Project Brain 加入认知层 → Handoff 关注传递成本 → MCL 关注"认知入口"而非内容量。推翻不是浪费，是在找"最小有效认知启动单元"。
